## [x] 6. Убрать хрупкую верстку: fixed width, float, inline-style и магические отступы

### Что за проблема простыми словами

Сейчас в проекте много верстки, которая держится на "магии":

- `width: 400px`
- `height: 400px`
- `margin-left: 46px`
- `float: right`
- `100vh`
- inline-style прямо в JSX

Это работает, пока контент короткий, экран удобный, а текст не меняется.

### Почему это реально проблема, а не придирка

Такой интерфейс очень легко ломается:

- на мобильном;
- при масштабировании браузера;
- при длинных текстах;
- при новых кнопках;
- при переводе интерфейса;
- при появлении ошибок в форме.

### Где это видно в проекте

- `src/Pages/AuthPage/authPage.module.css`
- `src/сomponents/form/form.module.css`
- `src/сomponents/layout/index.jsx`
- `src/сomponents/menu/index.jsx`
- `src/Pages/PersonalPage/index.jsx`

### Когда это станет большой проблемой

Когда дизайнер или менеджер попросит:

- добавить еще одно поле;
- показать текст ошибки;
- сделать нормально на ноутбуке и мобильном;
- увеличить шрифт;
- открыть страницу на другом масштабе.

Тогда "ровная" верстка начнет разъезжаться.

### Практический совет, как чинить

Базовые правила:

- вместо `float` использовать `flex` и `grid`;
- вместо фиксированных размеров чаще использовать `min()`, `max()`, `clamp()`;
- вместо `100vh` для полноэкранных блоков использовать `100dvh`;
- inline-style переносить в классы;
- не выравнивать элементы через случайные `margin-left`.

### Пример, как исправить

Было:

```css
.main {
  width: 400px;
  height: 400px;
  margin: 7em auto;
}
```

Лучше:

```css
.main {
  inline-size: min(100%, 28rem);
  min-block-size: min(32rem, calc(100dvh - 2rem));
  margin-inline: auto;
  margin-block: clamp(1rem, 6vh, 4rem);
  padding: clamp(1rem, 2vw, 1.5rem);
}
```

Для панели действий лучше так:

```css
.pageHeader {
  display: flex;
  align-items: center;
  justify-content: space-between;
  gap: 1rem;
  flex-wrap: wrap;
}
```

### Самопроверка

- [x] На ширине `320px` интерфейс не разваливается.
- [x] При масштабе браузера `200%` кнопки и поля не наезжают друг на друга.
- [x] В JSX почти не осталось inline-style.
- [x] Для выравнивания используются `flex`/`grid`, а не `float` и случайные `margin-left`.

## [ ] 9. Почистить структуру проекта и убрать опасные мелочи в именах

### Что за проблема простыми словами

В проекте есть имена, которые потом обязательно выстрелят:

- `servises` вместо `services`;
- `auidePage` вместо `guidePage`;
- папка `сomponents`, где первая буква выглядит как латинская `c`, но это кириллица;
- лишние или дублирующие зависимости.

### Почему это реально проблема, а не придирка

Это тот тип проблем, который долго выглядит безобидно, а потом внезапно:

- ломает импорт;
- путает IDE;
- мешает поиску по проекту;
- делает onboarding медленнее;
- приводит к "у меня работает, у тебя нет".

### Где это видно в проекте

- `src/servises`
- `src/Pages/auidePage`
- `src/сomponents`
- `package.json`

### Когда это станет большой проблемой

Когда проект попадет:

- в CI;
- к другому разработчику;
- на другую машину;
- в массовый рефакторинг.

Особенно опасна папка с кириллической буквой: визуально все похоже, а фактически это другой символ.

### Практический совет, как чинить

Делай это отдельной аккуратной задачей:

- переименовать папки и файлы в понятные английские имена;
- обновить импорты;
- удалить неиспользуемые зависимости;
- после каждого шага гонять `lint` и `build`.

### Пример, как исправить

Что стоит привести к норме:

- `servises` -> `services`
- `auidePage` -> `GuidePage`
- `сomponents` -> `components`

По зависимостям отдельно проверить, действительно ли нужны все пакеты из `package.json`.

### Самопроверка

- [ ] Во всех путях используются ожидаемые латинские имена.
- [ ] После переименования все импорты обновлены.
- [ ] `npm run lint` и `npm run build` проходят.
- [ ] В `package.json` не осталось явно лишних библиотек.
