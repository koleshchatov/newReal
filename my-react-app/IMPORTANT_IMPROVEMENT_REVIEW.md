## Проблемы

### 1. `fetchData` глотает ошибки -- это ломает весь смысл `unwrap()`

`src/Servisces/utils.js:26-44`

Ошибка создается через `throw new Error(...)`, но тут же ловится в `catch` и функция возвращает объект-заглушку. Thunk получает не исключение, а "успешный" результат. `.unwrap()` не срабатывает как должен.

На практике это значит: запрос упал с 500, а UI закрыл модалку и ведет себя как при успехе. Пункт 2 чеклиста не закрыт.

Заодно -- в коде остались `console.log`: `utils.js:32`, `role.service.js:14`.

### 2. `session.getDeviceId()` возвращает `null` на первом вызове

`src/auth/session.js:19-26`

Если `deviceId` еще нет в `localStorage`, код генерирует новый `v4()` и записывает его, но переменную `deviceId` не обновляет. Функция возвращает старое значение -- `null`. Первый логин уходит без реального `deviceId`.

### 3. Лишний пустой dialog на странице ролей

`src/Pages/RolePage/index.jsx:70-85`

Первый `ModalWindow` получает `open={modal.type}` -- строку вместо булева. Любое непустое `modal.type` открывает этот `dialog`, даже если внутри для текущего типа ничего нет. При `createRole` или `deleteRole` появляется пустая модалка поверх интерфейса. Это UI-баг, не придирка к стилю.

### 4. CSS все еще течет, верстка хрупкая

Пункты 5 и 6 чеклиста отмечены как закрытые, но по факту нет.

В `authPage.module.css` до сих пор глобальные селекторы `body`, `a`, `label` (строки 1-4, 8-10). В стилях фиксированные размеры, `margin-left`, `float`, `max-width: 100vh` (строки 38, 63, 85-86, 98-112). В `form.module.css` та же история (строки 49, 109, 140-164, 188, 200). В JSX много inline-style (`PersonalPage/index.jsx:41-48`, `Layout/index.jsx:9`).

Галочки в чеклисте по этим пунктам вводят в заблуждение.

### 5. Таблица пользователей: одно меню на все строки, `rowIndex` как key

`src/Pages/PersonalPage/index.jsx:21, 36-105`, `src/Components/Form/form.jsx:33-35`

Состояние `open` одно на всю таблицу, а не на строку. Ключ строки -- `rowIndex`. При сортировке или фильтрации меню поедет. Пункт 8 не начат.

### 6. Доступность почти не тронута

- `Button` (`button.jsx`) не принимает `title`, `aria-label`, `type`. В `Header` передается `title="Выйти"`, но `Button` его игнорирует.
- `Icon` (`icon.jsx`) рендерит `img` с children, у картинки нет `alt`.
- В логине `label` не связаны с `input` через `htmlFor`/`id` (`AuthPage/index.jsx:27-44`).
- Действия в меню пользователя -- кликабельные `li` вместо `button` (`PersonalPage/index.jsx:50-103`).

Пункт 7 не закрыт.

### 7. Опечатка в структуре, мертвые зависимости

Папка `Servisces` все еще с ошибкой. В `package.json` висят библиотеки, которых нет в исходниках: `@userfront/react`, `@userfront/toolkit`, `jsonwebtoken`, `react-jwt`, `react-uuid`, `v4`. Пункт 9 не закрыт.

## Как чинить

### 1. `fetchData`

Посмотри в `src/Servisces/utils.js`. Задай себе два вопроса: что возвращает `fetchData` при успехе? Что возвращает при ошибке? Потом пройди цепочку `fetchData -> service -> thunk -> component` для случая 401 или 500. На каждом шаге -- это `return` или `throw`?

Если запрос не прошел, ошибка должна долетать до thunk как исключение, а не как "обычный результат".

```js
if (!somethingOk) {
  throw new Error("...");
}
```

Не ловить ее раньше времени.

### 2. `deviceId`

`src/auth/session.js`. Посмотри на ветку, где значения еще нет. После записи нового UUID в `localStorage` -- что вернет функция в этой же попытке?

### 3. Модалки

`src/Pages/RolePage/index.jsx`. Подставь мысленно `modal.type = "createRole"` -- сколько `ModalWindow` получат truthy-значение? Сколько из них реально должны открыться?

### 4. CSS

По одному экрану за раз. Убери глобальные селекторы из `*.module.css`, потом самые опасные layout-хаки (`float`, фиксированные ширины). Проверь глазами. Потом следующий экран.

```css
.header {
  display: flex;
  gap: 12px;
}
```

Выравнивание через layout-механизм, а не через случайные `margin-left`.

### 5. Таблица

`src/Pages/PersonalPage/index.jsx`, `src/Components/Form/form.jsx`. Подумай: у пользователя есть поле, которое стабильно отличает его от других? Используй его как key вместо `rowIndex`. Меню действий должно принадлежать конкретной записи, а не таблице целиком. Что будет с открытым меню, если порядок строк поменяется?

### 6. Доступность

Начни с простого: если убрать картинку, останется ли у действия понятное имя? Если пройти форму только клавиатурой, будет ли ясно, какое поле для чего? Если элемент кликается -- это точно должен быть `li`, а не `button`?

```jsx
<label htmlFor="email">Email</label>
<input id="email" />
```

Связка "подпись -> конкретный инпут".

### 7. Структура и зависимости

Сначала через поиск посмотри, сколько файлов импортирует из `Servisces`. Потом реши: что безопаснее первым -- убрать мертвую зависимость или переименовать каталог. Переименование делай аккуратно и маленькими шагами, импорты ломаются легко.

Начинай с `fetchData` -- оно влияет на все запросы. Потом `deviceId` (мелкая правка). Потом модалки, потом таблица. CSS и доступность оставь на конец, там легко расползтись по всему проекту.
